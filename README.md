# Lonix: AI-Powered Indian Bank Loan Assistant

Lonix is a sophisticated, full-stack financial assistant designed to help users navigate the landscape of Indian bank loans. It combines real-time data fetching, advanced AI reasoning, and voice capabilities to provide a seamless, mobile-optimized experience aligned with RBI policies.

---

## 🚀 Key Features

### 1. AI Financial Expert
- **Powered by Gemini 3.1 Flash Lite**: Optimized for ultra-low latency and high-speed financial reasoning.
- **RBI Aligned**: Provides insights into repo rates, home loans, car loans, and personal loans based on current Indian banking standards.

### 2. Voice Assistant (STT & TTS)
- **Speech-to-Text**: Hands-free interaction using the browser's Web Speech API.
- **Text-to-Speech**: High-quality voice responses generated by the `gemini-2.5-flash-preview-tts` model.
- **Auto-Read Mode**: Toggleable setting to have Lonix automatically speak responses.

### 3. Advanced Chat History
- **Cloud Sync**: Securely saves your conversations to Firebase Firestore.
- **Search & Filter**: Find past advice using keywords or filter by date (Today, This Week, This Month).
- **Management**: Delete individual queries or clear your entire history.
- **Context Restoration**: Click any history item to bring it back into the active chat.

### 4. Interactive Financial Tools
- **EMI Calculator**: Real-time calculations with interactive Recharts visualizations (Principal vs. Interest).
- **Interest Rates Table**: Fetches the latest indicative rates from major Indian banks (SBI, HDFC, ICICI, etc.).

### 5. Mobile-First Design
- **Optimized UI**: Large touch targets, smooth spring animations (Framer Motion), and responsive overlays for calculators and tables.
- **Safe Area Support**: Designed to work perfectly on modern mobile browsers.

---

## 🧱 Tech Stack

- **Frontend**: React 19, Vite, Tailwind CSS 4, Framer Motion, Lucide Icons, Recharts.
- **Backend**: Node.js, Express.js (serving as a proxy and Vite middleware).
- **AI Services**: Google Gemini API (`@google/genai`).
- **Database & Auth**: Firebase Authentication (Google Login) and Firestore.

## 🛠️ Local Installation & Setup

Follow these steps to run Lonix on your local machine:

### 1. Prerequisites
- **Node.js**: Version 18 or higher.
- **NPM**: Standard package manager.
- **Google Gemini API Key**: Obtain one from the [Google AI Studio](https://aistudio.google.com/).
- **Firebase Project**: Create a project in the [Firebase Console](https://console.firebase.google.com/).

### 2. Clone and Install
```bash
# Clone the repository (or download the source)
cd lonix-ai

# Install dependencies
npm install
```

### 3. Environment Configuration
Create a `.env` file in the root directory and add your Gemini API key:
```env
GEMINI_API_KEY=your_gemini_api_key_here
```

### 4. Firebase Configuration
1. Enable **Google Authentication** in your Firebase project.
2. Create a **Firestore Database** (Enterprise Edition recommended).
3. Download your Firebase configuration and update `firebase-applet-config.json` in the root directory:
```json
{
  "apiKey": "...",
  "authDomain": "...",
  "projectId": "...",
  "appId": "...",
  "firestoreDatabaseId": "(default)"
}
```
4. Deploy the security rules provided in `firestore.rules` to your Firebase console.

### 5. Start Development Server
```bash
npm run dev
```
The application will be available at `http://localhost:3000`.

## 🏗️ Architecture: How it Works

### Voice Processing Pipeline
1. **Input**: The browser's `SpeechRecognition` API captures user voice and converts it to text in real-time.
2. **Reasoning**: The text is sent to the Express backend, which proxies the request to `gemini-3.1-flash-lite-preview`.
3. **Output**: The AI response is sent to the `gemini-2.5-flash-preview-tts` model. The raw PCM audio data is wrapped in a WAV header on the client side and played back using the Web Audio API.

### Full-Stack Integration
- **Express + Vite**: In development, Express uses Vite as middleware to provide Hot Module Replacement (HMR). In production, Express serves the static `dist` folder.
- **Firestore Security**: Rules are configured to ensure users can only read/write their own chat history, preventing unauthorized data access.

### Performance Optimizations
- **Thinking Level**: Set to `MINIMAL` for the chat model to ensure near-instantaneous responses.
- **Parallel Execution**: Chat history is saved to Firestore in the background while the AI response is being generated/spoken to the user.
